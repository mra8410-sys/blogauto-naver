네이버 블로그 자동화를 만드려고 한다.

프로그램은 codex.cmd를 사용해야 하므로 윈도우exe로 실행하도록 한다.

## UI 들어가야할 사항
 - Naver ID / Password
 - 주제
 - 카테고리
 - 키워드
 - 현재 작업상태 로그
 - 생성된 본문 영역
 - 생성된 이미지 미리보기 영역 (최대 4장 한정)
 - 기존 작업 조회 목록 및 성공, 실패 상태값

## 실행순서
 1. 사용자가 주제 및 키워드를 입력한다. (키워드는 입력하지 않을 수 있다 : 선택 값)
 2. 로컬에서 codex.cmd를 실행하여 codex를 실행한다.
 3. 내부적으로 사용자가 입력한 주제 및 키워드를 조합하여 NAVER 또는 GOOGLE 로 검색을 하여 정보를 검색한다.
 4. 검색된 목록의 상위 20개를 가져와, 해당 주제와 맞는지, 상품광고는 아닌지 체크한 후 사용을 할 수 있는 정보만 간추린다.
   - 그 후 제목을 생성한다.
   - 생성된 제목이 이미 등록한 제목인지 먼저 검색한다.
   - 제목은 cosine similarity를 사용하여 blog_history.jsonl 의 임베딩 값과 비교한다.
   - 최초 실행시 blog_history.jsonl이 없다면 그냥 다음 단계로 진행한다.
   - 유사도 0.75 미만이면 진행한다.
   - 유사도가 0.75 이상이면 해당 주제는 폐기하고, 2번 부터 다시 반복한다.
 5. 해당 페이지 내용을 AGENT가 분석하여 하나의 글을 작성한다.
   - 글은 반드시 NAVER 블로그 SEO에 최적화 되어야한다.
   - 글 작성시 최상단에는 위에서 생성한 제목을 먼저 붙인다.
   - 글자수는 1,500자~2,000자 내외로 한정한다.
   - 단, 무리하게 불필요한 말을 반복하거나 주관적인 문장을 넣어 글자수를 늘이는 꼼수는 쓰지 않는다.
   -1,500자 미만일 경우 해당 사유를 남긴 후 그대로 진행한다.
   - 글 생성시 이미지가 들어가면 좋은 위치에 GTP image 2.0 을 사용하여 이미지를 생성하며, 
     해당 이미지는 local 폴더의 \image\[블로그주제]_img_[SequenceNo].png 로 저장을 하고 이미지 삽입 위치를 [IMAGE INSERT - [SequenceNo]]로 삽입 위치를 알 수 있도록 글을 작성한다.
   - 이미지는 최대 4개 이하로 제한한다.
 6. 글 작성후 블로그 타이틀에 사용할 이미지를 생성한다.
   - 이미지는 GPT image 2.0 을 사용하여 만들며, 해당 본문 내용을 관통하는 이미지를 생성한다.
   - 이미지 스타일은 해당 본문과 어울리는 스타일을 일러스트 또는 카툰, 실사 형식 중 1가지를 골라 생성 한다.
   - 이미지 저장은 local 폴더의 \image\[블로그주제]_img_title.png로 저장한다.
 7. 웹브라우저를 호출하여 화면에 띄운다.
 8. 사용자가 UI에 입력한 Naver ID / Password 를 들고 https://naver.com 사이트 접속 후 직접 로그인을 한다.
   - 직접 마우스 및 키보드를 움직여 사람이 타이핑 하듯히 사람의 입력 속도로 입력을 하여야한다.
 9. 로그인 완료 시 https://blog.naver.com/[사용자id]/postwrite 로 접속을 하여 블로그로 글쓰기로 들어간다.
   - 가장 먼저 제목을 작성한다.
   - 그 다음 만들어 둔 타이틀 이미지를 먼저 삽입한다.
   - 그 다음 본문을 작성한다.
     > 본문글은 사람이 치는것 처럼 너무 빠르지 않도록 직접 타이핑 하듯이 작성해야한다.
   - 본문 작성중 [IMAGE INSERT] 태그가 있을때, 해당 IMAGE SequenceNo를 확인하여 저장된 동일한 SequenceNo를 가진 이미지를 삽입니다.
   - 이미지를 삽입할때는 반드시 위 아래 단락 사이에 여백이 있어야 한다.
 10. 모든 글이 작성되면 저장을 누른다.
 11. 저장이 완료되면 발행을 클릭한다.
   - 카테고리는 사용자가 입력한 카테고리와 동일한 카테고리로 선택한다.
   - 공개설정은 비공개로 한다.
   - 태그를 '#'을 붙여 최대 30개 미만으로 등록한다.
 12. 완료되면 jsonl에 기록한다.
   - 최초 jsonl이 없을 경우 blog_history.jsonl을 생성한다.
   - 들어갈 key값은, id, create_at, blog_id, title, topic, keyword, status, embedding_model, embedding 정보이다.
   - embedding을 사용해 들어갈 문장은 agent가 생성한 블로그 [제목] 이다. 이는 사용자가 입력한 주제가 아니다.

## 반드시 지켜져야할 사항
 1. 모든 입력은 사람이 입력하는 것 처럼 직접 마우스를 움직이고 키보드를 치는것 처럼 해야한다.
 2. 민감정보(PASSWORD)는 agent가 직접 읽어서는 안된다. 
 3. build 전에 블로그 작성시 필요한 dom 혹은 요소 정보가 필요할 경우 사용자에게 미리 묻거나 입력받을 수 있는 것을 생성한다.
 4. 최종 테스트 글쓰기를 완료하고 발행까지 되는것을 완료 단계로 본다.
